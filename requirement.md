项目需求：Pipeline & Asset Monitor（石油管道资产监控看板）
一、项目目标
构建一个 Web 应用，模拟展示油气管道沿线的资产（如阀门、泵站、传感器）位置和实时数据，体现完整的数据管道 + 后端 API + 前端地图可视化能力。

二、技术栈
前端：React + Leaflet（地图库）

后端：C# / ASP.NET Core（REST API）

数据管道：Python（数据生成/模拟 + ETL）

数据库：SQL Server（或 PostgreSQL）

调度：可选，如 Dagster 或简单的定时脚本

版本控制：Git + GitHub

三、功能需求
1. 数据管道模块（Python）
模拟生成油气资产数据（资产 ID、名称、类型（泵站/阀门/传感器）、经度、纬度、更新时间、温度、压力、状态等）。

定时（例如每分钟一次）生成随机的新读数（压力、温度变化）。

将生成的数据清洗、转换后存入 SQL Server 数据库（资产表 + 读数历史表）。

可选：用 Dagster 或 schedule 库实现简单调度。

2. 后端 API 模块（C# ASP.NET Core）
提供 REST API：

GET /api/assets – 返回所有资产列表（含经纬度、基本信息）。

GET /api/assets/{id}/readings – 返回指定资产的最新或最近几条读数（压力、温度、时间戳）。

GET /api/assets/{id}/history – 返回某资产的历史数据（用于图表）。

使用 Entity Framework Core 与 SQL Server 交互。

返回 JSON 格式数据，支持跨域（CORS）。

3. 前端地图可视化模块（React + Leaflet）
加载地图（默认视图显示模拟的管道沿线区域，例如加拿大阿尔伯塔省某坐标）。

调用 API 获取资产列表，在地图上绘制标记点：

不同资产类型使用不同图标（泵站、阀门、传感器）。

标记点悬浮/点击显示资产名称、最新读数和状态（正常/报警）。

点击标记后，侧边栏或弹窗中展示该资产的实时压力/温度仪表或折线图（模拟历史趋势）。

界面简洁，适配不同屏幕尺寸。

4. 文档与运行
提供 README.md，说明项目结构、如何配置数据库、如何运行后端和前端。

可选：提供模拟数据生成脚本的独立运行说明。

四、非功能要求
代码结构清晰，分层合理（前端组件化，后端多层架构）。

使用 Git 进行版本管理，提交信息规范。

如有时间，可添加简单的单元测试或 API 测试（非强制）。

六、注意事项
所有数据均为模拟生成，无需真实 PLC 或工业协议连接。

地图底图使用免费瓦片（如 OpenStreetMap），无需 API 密钥。

项目不强制要求 Docker 或云端部署。
